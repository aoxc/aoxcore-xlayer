# Security Policy: AOXCAN Sovereign AI Ecosystem

## 🛡️ Our Commitment
AOXCAN is built on the principle of **"Protocol Loyalty"** and **"Sovereign Security"**. As a hybrid ecosystem integrating AI (OKB-001) with the X-Layer blockchain, we maintain a zero-tolerance policy towards unmitigated vulnerabilities. Our 7-DAO (Sentinel) structure is designed to monitor, detect, and auto-repair threats in real-time.

## 🚀 Supported Versions
Only the latest versions of the AOXCAN Sentinel modules are supported for security updates.

| Version | Supported |
| ------- | --------- |
| v2.1.x (Surgical) | ✅ Yes |
| v2.0.x | ❌ No |
| < v2.0.0 | ❌ No |

## 🔍 Reporting a Vulnerability
We appreciate the work of independent security researchers. If you find a vulnerability, please follow these steps:

1. **Do not disclose publicly:** Please report the issue privately to protect the AOXCAN treasury and community.
2. **Report via Email:** Send a detailed report to **security@aoxcan.ai**. 
3. **Include Details:** Provide a Proof of Concept (PoC), the potential impact, and the Sentinel module affected (e.g., Treasury, Governance, Overseer).

## 💎 Bounty & Recognition
AOXCAN values ethical hacking. Reports that lead to a critical fix may be rewarded from the **AOXCAN Treasury** and recognized by the **Overseer DAO**, provided the researcher follows the responsible disclosure guidelines.

## 🚫 Out of Scope
- Social engineering or phishing against AOXCAN contributors.
- UI/UX bugs that do not compromise financial or neural integrity.
- Intentional manipulation of non-production testnets.
